Phase 1

Install the engine

Windows only — one download, one checkbox, ~3 min. Mac owners: skip this page.

Fintok · OpenLease
On a Mac? You are already done with Phase 1. macOS ships with Python — the engine OpenLease runs on — already installed. Nothing to download, nothing to click. Turn the page to Phase 2.
1

Download Python

In your web browser, go to the address below and click the big yellow Download Python button at the top. Don't worry about the version number — whichever one it offers is the right one.

↗ python.org/downloads
What you'll see: a file downloading, with a name like python‑3.13.exe, in your Downloads folder.
2

Run it — and tick the box

Open your Downloads folder and double‑click the file you just downloaded. An installer window opens with two buttons and a couple of checkboxes at the bottom.

This is the one thing you must not miss. At the bottom of that window, tick "Add python.exe to PATH" before you click anything else. It's a small checkbox and it is easy to skip — but without it, OpenLease can't find the engine and won't start.

Then click Install Now and let it run. If Windows asks "Do you want to allow this app to make changes?", click Yes.

3

Close it

When it says Setup was successful, click Close. That's the whole phase — no account, no restart, nothing to leave running.

You're done when: the installer says Setup was successful and closes. Python now sits quietly in the background forever; you will never open it or think about it again.

Engine installed.

Phase 2

Turn OpenLease on

Download, unzip, double‑click — it opens in your browser. ~5 min.

Fintok · OpenLease
1

Download OpenLease

In your browser, go to OpenLease's page on GitHub, click the green Code button, then Download ZIP.

↗ github.com/dweng1572179/openlease
What you'll see: a file named openlease-main.zip in your Downloads folder.
2

Unzip it

Open your Downloads folder and double‑click openlease-main.zip. A folder appears. Drag it onto your Desktop so it's easy to find — the folder's name doesn't matter, only where you can find it again.

3

Double‑click "Start OpenLease"

Open that folder on your Desktop. Right there — not in any sub‑folder — is the file for your computer. Double‑click it:

Mac Start OpenLease.command  ·  Windows Start OpenLease.bat
Mac may say "unidentified developer." If it blocks the file, right‑click it → Open → Open. You only do this once, ever.
What you'll see: a black window opens and shows progress. The first time takes a couple of minutes while it builds your tool — this is normal. Leave the window; after this it starts in seconds.
4

Log in

Your browser opens by itself to your new tool. If it doesn't within a minute, open your browser and type this into the address bar:

↗ localhost:8788

In the Password box type the starting password, then click the blue Sign in button:

Password changeme
Change it later: the password lives in a file called .env next to Start OpenLease, as OPENLEASE_PASSWORD — worth changing if this machine is reachable by anyone but you.
5

Load a few listings to try it on

OpenLease starts empty — real inventory comes in later, from a CSV you import, the crawler, or (in New York) the city's own vacancy feed. To have something to search right now, load 12 sample listings with one line. Open Terminal (Mac: Applications → Utilities) or Command Prompt (Windows: type cmd into the Start menu). Type cd with a trailing space, then drag your openlease folder from the Desktop into that window — the path fills itself in. Press Enter, then paste this and press Enter again:

$ .venv/bin/python -m app.seed
Windows .venv\Scripts\python.exe -m app.seed
What you'll see: seeded — 12 listings. That's it — three sample listings in each of New York, Miami and Los Angeles, plainly labeled "Demo Realty" so they're never mistaken for a real broker. This is the only line in this whole guide you type instead of click.

🔑 How to add any key — the same 3 clicks every time

  1. In OpenLease, click Settings — the small link in the very top‑right corner.
  2. You'll see a labeled box for each key. Paste your key into the box with the matching name, then click Save.
  3. The Settings page shows which capabilities are live next to what you just switched on. Done — no restart, ever.
None of these are required to search. The sample listings, a CSV you import, and the free government feeds already carry full address/size/rent with no key at all. Your first search

Try the exact search this guide is written around.

This isn't a hypothetical example — it's the search this whole guide, and the app's own README, describe. With the sample listings loaded (Phase 2, step 5) and no API key configured, do this:

1

Pick Miami

At the top of the OpenLease home page, set the metro to Miami.

2

Type the search

Into the search box, type exactly this, and press search:

Search retail in Wynwood ~1,500 SF under $8k/mo
3

Read the banner

$8,000/mo works out to a rent cap of about $64/SF/yr for a 1,500 SF space (8000 × 12 ÷ 1500). The one listing that otherwise matches — 2618 NW 2nd Ave, Wynwood — actually asks $95/SF/yr.

What you'll see: a banner across the results plainly saying the rent cap was relaxed to surface this listing, instead of either hiding zero results or quietly pretending it was a clean match. One result card, one pin on the map.
4

Open the listing

Click the result card. You'll land on a listing page with a Save button and a parcel panel — for this exact address it honestly reads No parcel matched rather than guessing. There's no Walk Score or Transit Score here yet: those come from a separate enrichment pass, which you run once with POST /api/enrich. Do that and this listing scores 86, with the per-category breakdown that explains it. Until you do, the row is simply absent — never a fake zero, because a zero is a real Walk Score (it means car-dependent) and you'd have no way to tell the two apart.

That's the whole promise, in one search.

Real listing, real map pin, an honest banner about what it had to relax to find it, and honest gaps instead of invented numbers where the data isn't there — and not one API key was used anywhere in this walkthrough.

Supply

Three ways listings get in.

In order of how much you should trust them — and what each one costs you in risk.

1. Free government feeds
No terms‑of‑service surface at all. NYC's Storefront Registry publishes a vacancy flag on every ground‑floor commercial space in the city — 43,978 vacant storefronts citywide as of this writing — via POST /api/import/storefronts. That's a lead source no broker site publishes.
2. Your own CSV
A broker export, a CoStar pull, whatever you already have rights to: POST /api/import/csv.
3. The crawler
POST /api/crawl, over the allowlist in app/data/sources.yml. Size and asking rent from a crawled page generally need the Anthropic key to read out of the page's prose — the government feed and your own CSV already arrive structured.
What the crawler will not do.

It never logs in. Not to any site, not ever, and there is no setting to turn that on — no account, no cookie, no registration‑ or NDA‑gated page. Getting past a bot‑wall on a page that's already public is one thing; crossing an authentication wall is another, and OpenLease doesn't do the second one, ever.

It stores facts, not expression. Address, size, ask, type, broker contact, and a link back to the original — never the broker's marketing paragraph verbatim (OpenLease writes its own description from the facts) and never a re‑hosted copy of their photo (it's hot‑linked from the broker's own server).

It identifies itself honestly and asks first. It reads and obeys robots.txt, including Crawl-delay, under the same identity the whole way through. One request every 3–5 seconds per domain, backing off if a site is struggling, a daily cap per domain, and it re‑uses what it already has rather than re‑asking every time.
Parcel data

What a metro doesn't publish, it says it doesn't publish.

Every metro has at least one field the public data simply does not carry. OpenLease shows that as a labeled gap, with the reason on hover — never a blank pretending to be a zero.

Los Angeles — owner name
LA County does not publish owner‑of‑record for free; California statute keeps it out of the open assessor feed.
Chicago — zoning (suburbs)
The zoning dataset is the City of Chicago's own layer — it's blank for anything in suburban Cook County, because the city doesn't zone land it doesn't govern.
Miami — zoning (incorporated cities)
Miami‑Dade's county zoning layer only covers unincorporated land. Inside cities like Wynwood and Brickell, OpenLease falls through to the municipal layer — and it's still blank wherever neither one publishes.
Why Chicago, if it isn't one of the three metros? Chicago's parcels, transit and scoring are fully built and tested — it's just not in the metro switcher. No Chicago brokerage the crawler checked publishes inventory outside a JavaScript search app, and OpenLease doesn't write per‑site scrapers, so a Chicago metro would show up empty. Import a Chicago CSV and it works today.
A tool that guesses is worse than a tool that admits it doesn't know. Every one of these reads "not published here," with the specific reason, rather than a silent blank or an invented value.

The safety net: a budget cap and a cache. Every paid call is checked against the Monthly paid‑spend cap you set on the Settings page — OpenLease refuses to spend past it. Rule of thumb: start with no keys — the sample data and any CSV or crawl you bring already work. Add Anthropic the moment you'd rather type a sentence than pick filters, and set the spend cap to whatever you're comfortable never exceeding.
