From Scratch.

You have a computer. That's it. By the end of this you'll have your own commercial‑leasing search running on it — built by you, owned by you, describing what it finds in plain English instead of a spreadsheet of filters.

There is almost no coding in here — one line, pasted once, to give it something to search. Every other step tells you exactly what to click and exactly what you should see afterwards. If a step doesn't look right, the last page tells you how to fix it.

Before we start

What you're actually building.

Read this page once. It's the only page with no clicking in it, and it makes everything else make sense.

Imagine a broker's assistant who never sleeps. You tell them, in a sentence, what you're after — "retail in Wynwood, about 1,500 square feet, under $8,000 a month." They go check every listing they're honestly allowed to read, plot the matches on a map, layer in free public data most tools charge for, and — if nothing quite fits — say exactly what they had to loosen to show you anything at all, instead of either an empty page or a quiet, dishonest match.

That assistant is what you're building. It's called OpenLease, and it will live on your computer and answer to nobody else.

Part 1

The engine

Windows only — Macs already have it. ~3 minutes.

Fintok · OpenLease
Do you have a Mac? Then Part 1 is already done. Every Mac comes with the engine built in. Don't download anything. Turn the page and start at Part 2.
1

Go and get Python

Open your web browser, click the address bar, and type this in:

↗ python.org/downloads

Click the big yellow Download Python button near the top. Ignore the version number — whichever one it offers is fine.

What you'll see: a file arriving in your Downloads folder, named something like python‑3.13.exe.
2

Open it — and tick the box

Double‑click that downloaded file. A grey installer window opens.

STOP. Do this before you click anything else. At the bottom of that window, tick "Add python.exe to PATH." It's tiny, easy to miss, and the single most common thing people get wrong — without it, OpenLease can't find the engine later.

Click Install Now. If Windows asks to allow changes, click Yes, then wait about a minute.

3

Close it and forget it

When it says Setup was successful, click Close.

You're done when: the installer closes. You will never open Python again — it just sits there being an engine.

Engine in. That was the hardest part.

Everything from here is downloading a folder and double‑clicking it.

Part 2

Get the tool

Download the folder, unzip it, put it on your Desktop. ~3 minutes.

Fintok · OpenLease
1

Download it

OpenLease lives on GitHub — a public shelf where code sits. Type this into your address bar, exactly as written:

↗ github.com/dweng1572179/openlease

Find the green "Code" button above the file list, click it, and click Download ZIP at the bottom of the menu.

What you'll see: a file called openlease‑main.zip in your Downloads folder. A "ZIP" is just a folder squashed small so it travels faster.
2

Unsquash it

Open Downloads and double‑click the ZIP.

On a Mac

Double‑click and you're done — the folder appears next to the ZIP.

On Windows

Double‑click it, then click Extract all at the top, then Extract. Windows will happily let you look inside a ZIP without unsquashing it, and OpenLease cannot run from in there.

What you'll see: a folder named openlease‑main. Everything in it is OpenLease.
3

Put it on your Desktop, and drop the "‑main"

Drag the openlease‑main folder onto your Desktop. Rename it to just openlease — click once, hit Enter (Mac) or F2 (Windows), type openlease, press Enter again.

The "‑main" is GitHub's, not yours. Renaming isn't required — the tool runs either way — but this guide says openlease throughout, and matching it means you never have to translate.
You're done when: a folder called openlease is on your Desktop, and opening it shows a pile of files — one called Start OpenLease. You can bin the ZIP now.

4

Load a few listings

OpenLease starts empty — on purpose. Real inventory comes later, from a CSV you import, the crawler, or (in New York) the city's own vacancy feed, and those are commands for someone comfortable with a terminal. To have something to search right now, open Terminal (Mac: Applications → Utilities) or Command Prompt (Windows: type cmd into the Start menu). Type cd with a trailing space, drag the openlease folder from your Desktop into that window so the path fills itself in, press Enter, then paste this and press Enter again:

$ .venv/bin/python -m app.seed
Windows .venv\Scripts\python.exe -m app.seed
What you'll see: seeded — 12 listings — three sample listings in each of New York, Miami and Los Angeles, all clearly labeled "Demo Realty." This is the one line of "code" in this entire guide.
5

Run the search this guide is written around

Back in the browser: set the metro to Miami, and type exactly this into the search box:

Search retail in Wynwood ~1,500 SF under $8k/mo

$8,000/mo works out to a rent cap of about $64/SF/yr for 1,500 SF (8000 × 12 ÷ 1500). The one listing that otherwise fits — 2618 NW 2nd Ave, Wynwood — actually asks $95/SF/yr. A strict filter would show you nothing.

What you'll see instead: one result, one pin on the map, and a banner saying plainly that the rent cap was relaxed to surface it. Click the card — the listing page shows a parcel panel that honestly reads No parcel matched for this address, and no Walk/Transit Score, because this sample skipped the separate scoring pass real listings go through. Gaps, not guesses.

Part 4

How it actually works

What happens when you search, and why it says "no" so often.

Fintok · OpenLease
The one idea behind everything below. A tool that guesses is worse than a tool that admits it doesn't know. Every rule on this page exists to keep OpenLease from quietly showing you something wrong.
1. robots.txt & sitemap
Before touching a broker's site, OpenLease reads its robots.txt (including Crawl‑delay) and asks under its own honest identity, the whole way through.
2. A structured feed
Some sites publish their own WordPress REST API or a JSON‑LD block on the listing page — address, broker, a link back. No scraping needed at all.
3. The page's own text
Most broker sites are only this: "1,500 SF," "$95/SF/yr," sitting in ordinary sentences. Reading that off the page is not a per‑site scraper — there is no CSS selector anywhere in this codebase — so it works on any site and a redesign costs nothing.
4. An LLM (only with a key)
For the rare page too unstructured for the first three rungs. An improvement, never a requirement — keyless, the first three rungs already carry size and rent, which is what a search like "under $8k/mo" needs to filter on at all.
What it will never do, no matter what. It never logs in — no account, no cookie, no registration‑ or NDA‑gated page, and there's no setting to turn that on. It stores facts, not expression: address, size, ask, broker contact, a link back — never the broker's marketing paragraph (OpenLease writes its own from the facts) and never a re‑hosted photo (hot‑linked from the broker's own server). This is the same fact‑vs‑expression line CoStar successfully argued against CREXi in court, and OpenLease stays well inside it on purpose.
Part 4, cont.

Fetching is slow on purpose

Why "get the listings" and "score them" are two separate steps.

Fintok · OpenLease
Crawl, then enrich
Fetching supply (POST /api/crawl) and scoring it for walkability (POST /api/enrich) used to run together. OpenStreetMap's free Overpass service rate‑limits hard, so a combined run spent 30 minutes backing off and never got past New York. Split apart, the same crawl finishes in nine minutes — Overpass is never touched while you're actually searching.
One request per neighborhood, not per listing
A 1.5‑mile circle of nearby amenities around each listing overlaps its neighbors' almost entirely once listings cluster. Asking once per listing made 345 requests for about 40 requests' worth of distinct data, and the free mirror started refusing them. OpenLease now asks once per ~3km tile and filters each listing back to its own radius — the scores come out identical, and a test asserts exactly that.
A geocoder is told the state, not just the address
A broker's feed is often national — one brokerage's listings span Texas, Florida and Manhattan. A geocoder scoped to New York does not decline a Florida street; it confidently returns a Brooklyn address with the same fake confidence as a correct hit. OpenLease reads the state off the listing's own URL before geocoding, so a state it doesn't cover is dropped — not silently mis‑pinned to the wrong city.
The scores, honestly labeled. Walk Score is Walk Score's own published 2011 methodology, computed here from OpenStreetMap and checked against their own published anchors: the Empire State Building scores 100 (they publish 100), Bay Ridge, Brooklyn scores 99 (they publish 98), and an industrial block in Vernon, LA scores 33 — proof the score actually discriminates rather than flattering every address. Transit Score is not calibrated — its normalization constant is an estimate, not fit against any published methodology — so treat it as a ranking, never a rating. The app says so on every listing page, not just here.
And where a metro simply doesn't publish something — LA won't give up an owner's name for free (state statute), Chicago's zoning layer is blank outside city limits, Miami's county zoning layer is blank inside Wynwood and Brickell — the field reads not published here, with the reason on hover. Never a blank pretending to be a zero.
Part 5

Give it eyes — the keys

Start with none of them. Everything in Parts 1–4 already works without one.

Fintok · OpenLease

🔑 What a "key" is, and how you add one

A key is a long password you copy from a provider's website and paste into OpenLease — it's how they know it's you asking. You pay them directly, at their price. OpenLease takes no cut.

  1. In OpenLease, click settings — small link, top‑right corner.
  2. Paste the key into the box with the matching name, then click Save. It applies immediately — no restart.
The one thing worth setting before you add a key: the spend cap. Same Settings page, Monthly paid‑spend cap. OpenLease checks every paid call against it and refuses to go over — set it to whatever you're comfortable never exceeding, before you paste anything.
Anthropic (Claude)
pay‑as‑you‑go, capped
Start here, if you add one at all. Type a sentence instead of picking filters, get a written reply and per‑listing chat, and let the crawler read size/rent off pages that don't publish them structurally. Sign up at console.anthropic.com, add billing before creating a key (a key with no credit behind it does nothing), then API Keys → Create Key → paste into Anthropic API key → Save.
Voyage AI
free tier is enough
Sharper ranking — matches "creative loft office" to a listing that never uses those words. Its free tier alone covers a corpus this size roughly 400× over. Sign up at dash.voyageai.com, no card needed, API Keys → Create → paste into Voyage API key → Save. Only ever changes ranking, never a hard filter.
Google Maps
one feature only
A Street View photo on the listing page — nothing else in OpenLease touches this key. Needs a billing‑enabled project at console.cloud.google.com with the Street View Static API turned on, then a restricted API key pasted into Google Maps key → Save.
Don't buy what you haven't already missed. Run the free version for a week — three metros, sample or imported listings, full parcel and Walk/Transit data — and add Anthropic the moment you catch yourself wishing you could just type a sentence.
When it goes wrong

Nothing here is broken.

Everything on this page has happened to somebody. Find your problem on the left, do the thing on the right. Nothing here can damage your computer or lose your work beyond what's noted.

The black window flashed up and vanished
Windows: Python isn't installed, or you missed the PATH checkbox. Reinstall from python.org/downloads and tick "Add python.exe to PATH" before clicking Install. Then double‑click Start OpenLease again.
Mac says "unidentified developer"
Expected. Right‑click the Start OpenLease file → Open → Open again. Once only, then never again.
The browser says "can't reach this page"
You got there before it did. Wait 30 seconds and refresh. Check the black window is still open — if you closed it, OpenLease turned off; double‑click Start OpenLease again.
The seed line says "command not found" or similar
Start OpenLease hasn't finished its first run yet — that's what creates the .venv folder the seed command lives in. Let the black window finish (watch for the "Starting OpenLease at..." line) before you open Terminal.
Search comes back with nothing, ever
No listings are loaded yet. Run the one‑line seed command from Part 3, step 4 — or, once you're past the demo, bring real supply in with a CSV import or the crawler (see the project's README for those commands).
My Anthropic key doesn't do anything
You almost certainly haven't bought credit yet. console.anthropic.com → Billing → add a card → buy credit. A key with no credit behind it does nothing.
I already run OpenProp on this computer
No conflict. OpenLease listens on 8788, OpenProp holds 8787 — both run at the same time, each in its own black window.
I want to turn it off
Close the black window — that's it. Or double‑click Stop OpenLease. Your listings, saves and portfolios are all in a local database file and will be there next time.
I think I've broken it beyond repair
Delete the openlease folder and start Part 2 again. You'd lose whatever listings, saves and portfolios were in that copy's database — nothing else on your computer is touched.

A little while ago that sentence would have sounded ridiculous. There is now a piece of real software running on your computer, that you set up yourself, that searches only what you point it at, and tells you the truth when it can't find a clean match.

It's private

It runs on your machine. Your searches, saves and portfolios never leave it — there's no company on the other end.

It's yours

No subscription, no seat fee, no expiring credits. You pay an AI provider directly, only for what you use, only if you choose to.

A near‑miss says what it relaxed. A missing field says why. An uncalibrated score says so. Nothing here quietly guesses.
